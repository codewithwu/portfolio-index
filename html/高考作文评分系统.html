高考作文评分系统

基于 LangGraph 构建的高考作文智能评分系统，采用多 Agent 协同架构，对作文进行四维度独立评估，支持流式输出与短路机制

Github仓库: https://github.com/codewithwu/langgraph-essay-grading
LangGraph Multi-Agent FastAPI SSE AI 教育
🎯

项目概述

  • 核心功能：输入作文题目与正文，返回四维度评分及综合得分
  • 评分维度：审题立意、论据分析、结构评估、语言文采
  • 技术架构：LangGraph 状态图工作流 + 条件路由
  • 应用场景：教师辅助批改、学生自评练习、AI 教育产品集成
⚙️

核心架构

  • 状态模型：EssayState (TypedDict) + ScoreDetail (Pydantic)
  • 工作流引擎：LangGraph StateGraph
  • 短路机制：relevance score ≤ 0.5 时跳过后续评估
  • 并行评估：fan_out 触发三 Agent 并行执行
START ──► check_relevance ──► (score>0.5?) │ Y → fan_out │ N → calculate_final_score │ ▲ └── fan_out ──► evidence ────────────┤ ├──► structure ─────────────┤ └──► expression ─────────────┘ │ calculate_final_score │ END
🤖

四维度独立评分 Agent

  • check_relevance：审题立意 — 考察是否准确理解题目
  • check_evidence：论据分析 — 考察材料是否充实
  • check_structure：结构评估 — 考察行文逻辑衔接
  • check_expression：语言文采 — 考察用词修辞句式
structured_llm = llm.with_structured_output(ScoreDetail) def check_relevance(state: EssayState) -> dict: messages = [ SystemMessage(content=build_json_prompt(ScoreDetail) + SYSTEM_PROMPT), HumanMessage(content=f"题目：{state['topic']}\n作文：{state['essay']}"), ] result = structured_llm.invoke(messages) return {"relevance": result}
⚡

短路机制（Short-Circuit）

  • 触发条件：relevance.score ≤ 0.5（严重偏题）
  • 节省开销：跳过 75% 的 LLM API 调用
  • 防止虚高：避免跑题作文因后续得分而虚高
  • 实现方式：add_conditional_edges + 自定义路由函数
def route_relevance(state: EssayState) -> str: if state["relevance"].score > RELEVANCE_THRESHOLD: return "fan_out" return "calculate_final_score"
🔀

Fan-Out 并行评估

  • 并行节点：check_evidence / check_structure / check_expression
  • 汇聚点：统一汇合至 calculate_final_score
  • 实现方式：add_edge 从同一节点指向多个节点
  • 优势：三个维度同时评估，提升效率
workflow.add_edge("fan_out", "check_evidence") workflow.add_edge("fan_out", "check_structure") workflow.add_edge("fan_out", "check_expression")
📊

维度权重配置

  • 审题立意：30% — 最重要维度
  • 语言文采：30% — 与立意并列最重要
  • 论据分析：20% — 材料充实度
  • 结构评估：20% — 逻辑与衔接

流式 SSE 输出

  • 实时推送：逐节点推送评分结果
  • 前端渲染：SSE 事件实时更新 DOM
  • 无需等待：节点完成立即显示评分卡片
  • 骨架屏：加载中动态文案提示
event: check_relevance data: {"node": "check_relevance", "update": {"relevance": {"score": 0.90, "reason": "..."}}} event: check_evidence data: {"node": "check_evidence", "update": {"evidence": {"score": 0.85, "reason": "..."}}}

关键设计模式

  • TypedDict 状态：部分状态更新，LangGraph 自动合并
  • Structured Output：llm.with_structured_output(PydanticModel)
  • 条件路由：add_conditional_edges 实现动态流程
  • 扇出-扇入：fan_out 单点触发多节点并行
  • 异步桥接：threading.Thread + queue.Queue 桥接同步流
  • 单例+工厂：@lru_cache + LLMFactory
