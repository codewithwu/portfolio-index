LangGraph 智能客服分流 Agent

基于 LangGraph 构建的智能客服系统，集成问题分类、情感分析、自动路由与人工升级机制，配套 FastAPI 后端接口和现代化 Web 聊天界面

核心架构：LangGraph 工作流

  • 状态模型：扁平化 TypedDict 设计
  • 状态字段：query、category、sentiment、response
  • 节点数量：6 个工作流节点
  • 路由逻辑：情感优先级最高
START → categorize → analyze_sentiment → route_query ↓ ┌─────────────────┼─────────────────┐ ↓ ↓ ↓ [消极] escalate [技术支持] handle_tech [账单] handle_bill ↓ ↓ ↓ └─────────────────┼─────────────────┘ ↓ END
🔧

六个工作流节点

  • categorize：LLM 分类（技术支持/账单/一般）
  • analyze_sentiment：LLM 情感分析（积极/中性/消极）
  • handle_technical：技术支持专业回复
  • handle_billing：账单问题专业回复
  • handle_general：一般咨询回复
  • escalate：硬编码人工升级消息
🔀

条件路由逻辑

  • 优先级判断：情感 > 类别
  • 消极情绪：直接升级人工客服
  • 技术支持：路由至 handle_technical
  • 账单问题：路由至 handle_billing
  • 一般咨询：路由至 handle_general
def route_query(state): if state["sentiment"] == "消极": return "escalate" elif state["category"] == "技术支持": return "handle_technical" elif state["category"] == "账单问题": return "handle_billing" else: return "handle_general"

JSON Agent（结构化输出）

  • 设计目标：严格符合 Pydantic Schema
  • 重试机制：最多 3 次验证重试
  • Schema 基类：JsonAgentSchema
  • 核心字段：is_valid: bool
class JsonAgent: def invoke_with_validation(self, input, max_retries=3): for attempt in range(max_retries): result = self.agent.invoke(...) is_valid, parsed = self.parse_json_response(result) if is_valid: return parsed raise ValueError("JSON 响应验证失败")
