LangGraph Human-in-the-Loop SQL Agent FastAPI 状态机

SQL Agent LangGraph

基于 LangGraph 的智能 SQL 查询代理系统

Github仓库: https://github.com/codewithwu/sql-agent-langgraph
📌
项目定位
  • 智能 SQL 查询代理，支持自然语言转 SQL
  • 语法验证、人工审核、结果智能汇总
  • 采用状态图编排范式，安全可控的 human-in-the-loop
🛠️
技术栈
  • 编排框架：LangGraph 状态图编排
  • LLM 集成：LangChain Core 多 provider
  • 后端框架：FastAPI 高性能异步
  • 数据库：SQLDatabase 多方言支持
📊
目录结构
  • sql_agent/：SQL Agent 核心模块
  • langchain_agent/：LangChain 集成工具库
  • src/：FastAPI 后端服务
  • Chinook.db：示例 SQLite 数据库

🔧
AgentState 状态定义
  • 继承 MessagesState，自动管理消息历史
  • generated_num：SQL 生成次数（上限 3 次）
  • generated_query：生成的 SQL 语句
  • is_query_valid：语法验证结果
📝
节点函数 (nodes.py)
  • generate_query_node：调用 LLM 生成 SQL
  • check_query_node：结构化输出验证，最多 3 次
  • human_review_node：interrupt() 暂停等待
  • run_sql_node：调用 sql_db_query 执行
🔀
条件路由 (router.py)
  • generated_num > 3 → summarize_result
  • is_query_valid == True → human_review
  • human_action == approved → run_sql
  • human_action == rejected → generate_query
🔌
Human-in-the-Loop 机制
  • interrupt()：暂停图执行，等待人工决策
  • Command(resume={...})：外部恢复执行
  • 支持 approve / reject 双路径
  • 审批时可编辑 SQL 语句
🏭
LLMFactory 工厂模式
  • 支持 8+ 种 LLM 提供商
  • @lru_cache() 单例模式
  • providers: zhipu, bailing, siliconflow, modelscope, longcat, deepseek 等
🛡️
中间件设计
  • @wrap_model_call：拦截模型调用
  • @wrap_tool_call：工具过滤
  • 动态模型选择、工具过滤、PII 处理
  • Token 限制自动摘要、失败自动切换
📡
API 端点
  • POST /api/v1/chat/sessions：创建会话
  • GET /api/v1/chat/sessions/{thread_id}：获取状态
  • POST /approve：审批通过
  • POST /reject：拒绝重生成
